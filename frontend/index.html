🚀
Getting Started

        

          
Spektra HackOps automates provisioning and managing hackathon environments on Microsoft Azure & Entra ID. It handles user creation, licensing, RBAC assignments, OneDrive file uploads, lifecycle automation (read-only transitions and cleanup), cost tracking, and more.

          

            

              
Step 1

              
Connect

              
Enter your SPN credentials on the Connection page and click Test Connection.

            

            

              
Step 2

              
Verify Permissions

              
Go to Tenant Check and confirm your SPN has the required Graph API permissions.

            

            

              
Step 3

              
Follow the Lifecycle

              
Use the sidebar steps 01–06 in order: Setup → RBAC → Upload → Manage → Read-only → Cleanup.

            

          

        

      


      
      

        
🧭
Navigating the App

        

          

            
SidebarThe left panel organizes all views: Overview (Connection & Dashboard), Lifecycle Steps (01–06), Tools, and AI Chat / Help. Status dots next to lifecycle steps show progress. On mobile, tap ☰ to open.

            
Top barShows the Active Hack selector — pick a saved hack and all views auto-fill with its prefix, dates, and config. The Activity button opens a feed of recent operations.

            
DashboardBird's-eye view of all hacks — active and archived. See user counts, lifecycle phase, countdown timers, and per-hack cost estimates.

            
Connection dotThe green/red dot in the sidebar footer shows real-time connection state. Updates as you type credentials. Click it to jump back to Connection.

          

        

      


      
      

        
🔗
Connection

        

          
The Connection page is where you enter your Azure Service Principal (SPN) credentials. These are used for all Graph and Azure ARM API calls. Credentials are sent per-request and never stored on the server.

          

            
Tenant IDYour Entra directory ID (GUID). Found in Azure Portal → Entra ID → Overview.

            
Client IDThe Application (client) ID from your App Registration.

            
Client SecretThe secret value (not the secret ID) from Certificates & secrets.

            
RememberOptionally save Tenant/Client ID in localStorage. Secret storage is optional but not recommended on shared machines.

            
Test ConnectionValidates credentials via the Graph API. Shows your tenant's primary domain on success.

          

          
💡 If saved credentials exist, the app auto-redirects to Dashboard on load. Change credentials anytime via the sidebar Connection link or the footer dot.

        

      


      
      

        

          01📋
Setup Hack

        

        

          
Create a new hackathon environment — provisions users, groups, licenses, TAPs, and optionally RBAC roles in one operation.

          

            
Hack PrefixShort identifier (e.g. nyc-esri-gcc-) prepended to all users/groups. Used to discover and manage resources later.

            
ModeTeam-based: Creates team groups with users distributed. Flat: Users without grouping.

            
Teams / UsersControls team count and users per team. Total = teams × users per team.

            
Admin UsersNumber of admin accounts. Admins get Global Reader role and their own admins group.

            
LicensesM365/Azure licenses to assign. Load from Tenant shows available SKUs with remaining capacity.

            
TAP LifetimeTemporary Access Pass duration in minutes. 0 = tenant maximum.

            
PasswordOptional shared password. Blank = unique random per user. "Reuse last" restores previous hack's password.

            
Hack NameStamped as companyName on each user for easy identification.

            
Lifecycle DatesStart, Hack, Read-Only, Delete dates. Read-only and delete auto-schedule lifecycle jobs.

            
Pre-flightValidates permissions, license availability, and subscription access before provisioning.

            
ScheduleDefer provisioning to a future date/time instead of running immediately.

          

          
💡 After provisioning, lifecycle jobs are automatically scheduled if read-only and delete dates are provided.

        

      


      
      

        

          02🔐
Azure RBAC

        

        

          
Assign Azure RBAC roles (Owner, Contributor, or Reader) on one or more subscriptions to hack team groups or individual users.

          
Discovers principals by hack prefix and assigns the selected role on each subscription. The interactive picker shows subscription names, IDs, and states.

          
⚠️ Requires: The SPN must have Owner or User Access Administrator role on target subscriptions.

        

      


      
      

        

          03📁
Upload Files

        

        

          
Upload files or folders to every user's OneDrive in bulk. Supports drag-and-drop, folder selection, and preserves folder structure.

          

            
UsersEnter UPNs (one per line) or click "Load from tenant" to fetch hack users.

            
FilesDrag & drop files/folders or use the file picker. Folder structure is preserved.

            
DestinationRoot-relative path in each user's OneDrive (e.g. /Documents/HackFiles).

            
ConcurrencyParallel uploads. Higher = faster but may hit API throttling.

            
Dry RunPreview the upload plan without uploading.

            
License CheckVerify users have SharePoint Online (OneDrive) licenses before uploading.

          

          
💡 Upload supports files up to 250 MB using resumable sessions.

        

      


      
      

        

          04⚙️
Manage & Report

        

        

          
Load and manage a previously provisioned hack. Active Hacks for live operations; Archived Hacks for historical reports.

          

            
Load HackEnter prefix or select from dropdown. View users, passwords, TAPs, licenses, and groups.

            
Regenerate TAPNew Temporary Access Passes for users with expired TAPs.

            
Reset PasswordReset passwords for one or more hack users.

            
Assign LicensesAdd missing licenses. Admins excluded by default; toggle to include.

            
GitHub EMUEnable GitHub Enterprise Managed User access for participants.

            
ReportCost and usage summaries — per subscription, team, and user. Works for active and archived hacks.

            
Drift DetectionCompare saved state against live Entra to detect changes made outside this tool.

            
Expand HackAdd more teams or users to an existing hack without re-provisioning.

            
Set End DateSchedule automatic read-only and cleanup lifecycle jobs.

            
ExportDownload results as Excel (.xls) or CSV.

          

        

      


      
      

        

          05🔒
Read-only Mode

        

        

          
Freeze participant access when submissions close. Downgrades RBAC from Owner/Contributor to Reader, so judges can review but participants can't change resources.

          
Two modes: team (discovers groups) or flat (discovers individual users). Preview mode shows what will change before executing.

        

      


      
      

        

          06🧹
Cleanup

        

        

          
Completely remove all resources created for a hack:

          
    
            
  • ✓ Delete all users matching the hack prefix
    • 
            
  • ✓ Delete all groups (team groups + admin group)
    • 
            
  • ✓ Remove all RBAC role assignments on specified subscriptions
    • 
            
  • ✓ Archive state data for future reports (passwords & TAPs stripped)
    • 
          

          
🚨 Warning: Cleanup is irreversible. All users, groups, and permissions are permanently deleted. A confirmation dialog shows exactly what will be removed.

📖
Glossary

        

          

            
SPNService Principal — the identity for API calls. Created via App Registration in Entra ID.

            
UPNUser Principal Name — the sign-in name (e.g. user01@contoso.onmicrosoft.com).

            
TAPTemporary Access Pass — a time-limited passcode for initial sign-in without a password.

            
RBACRole-Based Access Control — Azure's permission system for subscriptions and resources.

            
SKUStock Keeping Unit — a license plan identifier (e.g. ENTERPRISEPACK = Office 365 E3).

            
EMUEnterprise Managed Users — GitHub accounts managed through Entra ID via SCIM.

            
Hack PrefixShort string prepended to all users/groups, making them easy to discover and clean up.

            
DriftDifference between saved state and live Entra — changes made outside this tool.

🛡️
Security Best Practices

        

          
    
            
  • 🔑 Use least-privilege app permissions — only grant what each lifecycle action needs.
    • 
            
  • 🔄 Rotate the client secret after each hack event.
    • 
            
  • 🚫 Avoid saving the client secret in localStorage on shared machines.
    • 
            
  • 📋 Run Tenant Check before your first hack to verify permissions.
    • 
            
  • 🧹 Always run Cleanup after the event to remove all provisioned resources.
    • 
            
  • 📊 Use Drift Detection to catch unexpected changes during the event.
