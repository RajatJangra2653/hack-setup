❓ Help & Documentation

      
Complete guide to all features of the Spektra Hack Setup tool.


      

        
🏠 Overview

        

          
Spektra Hack Setup is a comprehensive tool for provisioning and managing hackathon environments on Microsoft Azure & Entra ID. It automates user creation, licensing, RBAC assignments, file uploads to OneDrive, lifecycle management (scheduling read-only and cleanup), and more.

          
The tool uses a Service Principal (App Registration) to interact with Microsoft Graph API and Azure ARM API. Enter your SPN credentials at the top of the page — they are used for all operations and never stored on the server.

        

      


      

        
🛠 Setup Hack

        

          
The primary tab for creating a new hackathon environment. Provisions users, groups, licenses, TAPs, and RBAC roles in one go.

          

            
Hack PrefixA short identifier (e.g. nyc-esri-gcc-) prepended to all user and group names. Used to discover and manage hack resources later.

            
ModeTeam-based: Creates team groups (team01, team02, …) with users distributed across them. Flat: Creates all users without team grouping.

            
Teams / Users per teamControls how many teams and users per team to create. Total users = teams × users per team.

            
Admin usersNumber of admin accounts to create. Admins are assigned Global Reader role and placed in an admins group.

            
LicensesSelect M365/Azure licenses to assign to participants. Enable “Assign selected M365 licenses to admin users also” when admins need Teams, SharePoint/OneDrive, Copilot, Power BI, or Office apps. Click "Load from Tenant" to see available licenses with remaining capacity.

            
TAP lifetimeTemporary Access Pass duration in minutes. Set to 0 to use the tenant's maximum. Users use TAP to sign in without a password.

            
PasswordOptional shared password for all users. Leave blank for unique random passwords per user.

            
Force password changeIf checked, users must change their password on first sign-in.

            
Hack nameStamped as companyName on each user object for easy identification.

            
Lifecycle datesHack Start Date, Hack Date, Read-Only Mode Date, and Delete Date are stored with the hack. Read-only and delete dates schedule lifecycle jobs automatically when provided.

            
Skip existingIf checked, skip users that already exist in the tenant instead of failing.

            
Pre-flight checkValidates permissions, license availability, and subscription access before provisioning.

            
Schedule HackSchedule provisioning for a future date/time instead of running immediately.

          

          
💡 Tip: After provisioning, the tool automatically schedules read-only and cleanup jobs if lifecycle dates are provided.

        

      


      

        
🔐 Assign Permissions

        

          
Assign Azure RBAC roles (Owner, Contributor, or Reader) on one or more Azure subscriptions to the hack's team groups or individual users.

          
The tool discovers principals (groups/users) by hack prefix and assigns the selected role on each specified subscription.

          
⚠️ Requires: The SPN must have Owner or User Access Administrator role on the target subscriptions.

        

      


      

        
📤 Upload Files

        

          
Upload files or folders to every user's OneDrive in bulk. Supports drag-and-drop, folder selection, and maintains folder structure.

          

            
UsersEnter UPNs (one per line) or click "Load from tenant" to fetch all users.

            
FilesDrag & drop files/folders or use the file picker. Folder structure is preserved.

            
DestinationRoot-relative path in each user's OneDrive (e.g. /Documents/HackFiles).

            
ConcurrencyNumber of parallel uploads. Higher = faster but may hit API throttling.

            
Dry runPreview the upload plan without actually uploading anything.

          

          
💡 Tip: Check licenses first to ensure users have OneDrive (SharePoint Online) enabled.

        

      


      

        
👀 Read-Only Mode

        

          
Downgrade all hack users' Azure subscription access to Reader role. This is useful when the hack ends and you want to prevent further changes while keeping read access for review.

          
Enter the hack prefix and subscription IDs. The tool discovers all principals (groups/users) by prefix and downgrades their RBAC assignments from Owner/Contributor to Reader.

          
Two modes: team (discovers groups) or flat (discovers individual users).

        

      


      

        
🗑️ Cleanup Hack

        

          
Completely remove all resources created for a hack. This includes:

          
    
            
  • Delete all users matching the hack prefix
    • 
            
  • Delete all groups (team groups + admin group)
    • 
            
  • Remove all RBAC role assignments on specified subscriptions
    • 
            
  • Move blob state data into Archive for future reports
    • 
          

          
🚨 Warning: This action is irreversible. All hack users, groups, and permissions will be permanently deleted. The saved hack record is retained under Archive without passwords or TAPs.

        

      


      

        
📋 Manage Hack

        

          
Load and manage a previously provisioned hack from Azure Blob Storage. Use Active Hacks for operations and Archived Hacks for historical reports after cleanup.

          

            
Load HackEnter the hack prefix to load state from blob storage. View all created users, their passwords, TAPs, licenses, and group memberships.

            
Regenerate TAPGenerate new Temporary Access Passes for users whose TAPs have expired.

            
Assign LicensesAssign additional licenses to hack users who are missing them. Admin users are excluded by default and can be included with the admin checkbox.

            
ReportGenerate a summary of created users/admins, assigned licenses, and cost allocation by subscription, team, and user. Reports work for active and archived hacks.

            
Set End DateSchedule automatic read-only and cleanup jobs for the hack.

            
ExportDownload results as Excel (.xls) or CSV for distribution to hack participants.

          

        

      


      

        
📅 Scheduler Tab

        
Create and manage scheduled provisioning, read-only, and cleanup jobs. Jobs run automatically at the scheduled time.

        

          

            
Create JobSchedule a provision, read-only, or cleanup job for a future date/time. Provision config is inherited from Setup Hack tab.

            
Filter JobsFilter the jobs list by status (pending, running, completed, failed, cancelled).

            
Cancel JobCancel a pending job before it executes.

📖 Glossary

        

          

            
SPNService Principal — the identity used to authenticate API calls. Created via App Registration in Azure AD.

            
UPNUser Principal Name — the sign-in name of a user (e.g. user01@contoso.onmicrosoft.com).

            
TAPTemporary Access Pass — a time-limited passcode that can be used instead of a password for initial sign-in.

            
RBACRole-Based Access Control — Azure's system for managing permissions on subscriptions and resources.

            
SKUStock Keeping Unit — an identifier for a Microsoft license plan (e.g. ENTERPRISEPACK = Office 365 E3).

            
Hack PrefixA short string prepended to all users/groups for a hack, making them easy to discover and clean up.
